# Contributing to addypowerwash2

Thank you for your interest in contributing to addypowerwash2! We appreciate your support and want to make the contribution process as smooth as possible.

## Code of Conduct

This project adheres to a Code of Conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior to the project maintainers.

## How to Contribute

### Reporting Bugs

Before creating a bug report, please check if the issue already exists. When creating a bug report, include:

- **Clear description** of what the bug is
- **Steps to reproduce** the behavior
- **Expected behavior** vs actual behavior
- **Screenshots or logs** if applicable
- **Your environment** (OS, browser, version, etc.)

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, please include:

- **Clear description** of the enhancement
- **Why this would be useful**
- **Possible implementation** (optional)

### Pull Requests

1. Fork the repository and create your branch from `main`
2. If you've added code, add tests
3. Ensure your code follows the project's style guidelines
4. Write a clear commit message
5. Push to your fork and submit a pull request

## Development Setup

1. Clone your forked repository
2. Install dependencies: `npm install`
3. Create a feature branch: `git checkout -b feature/YourFeature`
4. Make your changes
5. Test your changes: `npm test`
6. Commit and push your changes
7. Submit a pull request

## Style Guidelines

- Use consistent indentation (2 spaces)
- Write meaningful commit messages
- Comment complex code sections
- Follow existing code style
